# MyButton 项目文档

## 项目概述

MyButton 是一个 Nuke 插件工具管理系统，提供工具搜索、安装、收藏、元数据编辑等功能。项目采用 PySide6/PySide2 GUI 框架，支持 SQLite 数据库存储用户数据，实现了多用户隔离和智能缓存机制。

#### 11. `window_01/search_window.py`
**功能**：搜索型工具窗口主类

**说明**：
- `SearchToolWindow` 类：全透明无边框窗口
- **虚拟列表技术**：固定 15 个按钮槽位（3行×5列），动态更新内容，避免大量组件创建
- **多条件搜索**：
  - `C=:` - 类名（精确匹配 matchClass）
  - `L=:` - 标签（模糊匹配 tag）
  - `P=:` - 路径（模糊匹配 main_file）
  - `N=:` - 名称（模糊匹配 name）
  - 普通关键词 - 模糊搜索所有字段
- **智能缓存机制**：LRU 缓存，最多 100 个搜索结果，避免重复数据库查询
- **滑块分页浏览**：垂直滑块控制显示区域，支持鼠标滚轮
- **四个拖拽边框**：低调深灰色（RGBA 20,23,20,128），支持窗口拖拽移动
- **仅已安装过滤**：复选框控制，默认开启
- **Debounce 搜索**：300ms 延迟执行，避免频繁查询
- **透明度参数**：
  - 按钮区域背景：alpha=10
  - 搜索框背景：alpha=30
  - 拖拽边框：alpha=128
- **窗口定位**：可配置是否在显示时定位到鼠标位置（ShowP）
- **快捷键**：Esc 或 A 键关闭窗口

**依赖**：
- `qt_imports` - Qt 框架
- `window_01.config` - 配置
- `window_01.db` - 数据库查询
- `window_01.widgets` - 工具按钮组件

### 📁 数据管理

#### 15. `sqlite_file_setup.py`
**功能**：SQLite 数据库设置和管理

**说明**：
- **用户数据库隔离**：每个用户独立数据库（tools_{uuid}.db）
- **数据库初始化**：创建两个表
  - `tools_index` - 工具索引表（id, name, tag, matchClass, main_file, icon_file, help_file, category, has_icon, has_help）
  - `user_tools` - 用户配置表（tool_id, is_installed, is_favorite, usage_count, is_enabled, custom_tag, last_used）
- **索引优化**：为 name, tag, matchClass, category, tool_id 创建索引
- **工具同步策略**：
  1. 保存用户配置（安装/收藏状态、使用次数）
  2. 清空 tools_index 表
  3. 从 JSON 文件重新插入所有工具
  4. 恢复用户配置（INSERT OR REPLACE）
  5. 清理已删除工具的配置
- **用户配置保存**：安装/收藏状态、使用次数、最后使用时间
- **数据库切换**：支持多数据库 UUID 切换
- **搜索工具**：支持多条件组合查询（keyword, tag, matchClass, category, is_installed, is_favorite）
- **工具状态更新**：安装状态、使用次数、收藏状态切换

**依赖**：
- `sqlite3` - SQLite 数据库
- `paths_setup` - 路径管理
- `toolsfile_json` - JSON 文件操作
- `uuid` - UUID 生成
- `getpass` - 获取用户名

**被依赖**：
- `settool.py` - 主应用调用
- `tool_list_manager.py` - 列表管理调用
- `metadata_editor.py` - 元数据编辑调用

## 数据流

### 1. **工具扫描流程**：
```
toolsfile_json.scan_tool_files() 
→ 递归扫描 tools/ 目录（排除 # 号文件夹）
→ 识别 .py/.nk 主文件
→ 识别配套文件 (.json, .png, .html/.md)
→ ensure_tool_json() 创建/更新元数据
  → 生成 UUID（如果不存在）
  → 补充缺失字段（tag, matchClass, files）
  → 写入 JSON 文件
→ 返回工具列表
```

### 2. **数据库同步流程**：
```
sqlite_file_setup.sync_tools_to_database()
→ 保存用户配置（SELECT user_tools）
→ 清空 tools_index 表（DELETE）
→ 扫描 JSON 文件（toolsfile_json.get_all_tools）
→ 插入所有工具到 tools_index
→ 恢复用户配置（INSERT OR REPLACE）
→ 清理已删除工具的配置
→ 提交事务
```

### 3. **搜索流程**：
```
用户输入搜索文本
→ _parse_search_text() 解析参数（C=:, L=:, P=:, N=:, 关键词）
→ 构建缓存键（搜索参数 + 安装过滤状态）
→ 检查 SmartCache 缓存
  → 缓存命中：直接返回结果
  → 缓存未命中：
    → FastDBQuery.search_tools() 数据库查询
    → INNER JOIN user_tools（支持安装过滤）
    → 排序：收藏优先，按名称排序
    → 缓存结果（LRU 策略，最多 100 个）
→ 虚拟列表显示（固定 15 个按钮槽位）
→ 滑块控制分页浏览
```

### 4. **工具执行流程**：
```
点击工具按钮
→ nukendoesget.getcurnodes() 获取选中节点
→ 构建工具完整路径（toolbox_path + tpath）
→ nuke.execute_tool(tool_path, nodesnames_list)
  → .py 文件：exec 执行，注入 selected_nodes_list
  → .nk 文件：nuke.nodePaste()
  → 其他：nuke.tcl('source')
→ 隐藏搜索窗口
```

### 5. **元数据编辑流程**：
```
用户修改元数据（名称/标签/匹配类）
→ MetadataEditor 更新数据库（UPDATE tools_index）
→ 更新 JSON 文件（ReadWriteJson.WriteJson）
→ 更新内存数据（current_tool_data, all_tools）
→ 刷新 UI 显示
```

### 6. **图标管理流程**：
```
从剪贴板获取图标
→ QApplication.clipboard().mimeData()
→ 支持 QImage 或文件 URL
→ 预览缩略图（100x100）
→ 保存图标（工具名.png）
→ 更新数据库（UPDATE tools_index SET icon_file）
→ 更新 JSON 文件（files.icon）
→ 更新内存数据
```

### 7. **帮助文档处理流程**：
```
加载 Markdown 文件
→ 提取图片引用（正则匹配 ![alt](path) 和 <img src="path">）
→ 复制图片到 pnghelp 目录（版本控制：_v1, _v2...）
→ 更新 Markdown 中的图片路径为 pnghelp 文件名
→ 保存 Markdown 到工具目录
→ 转换为 HTML（markdown 库或简单转换）
→ 替换图片路径为 file:/// 绝对路径
→ 显示在 QTextBrowser
```

## 核心设计模式

1. **虚拟列表**：固定按钮槽位（15个），动态更新内容，避免大量组件创建，提升性能
2. **智能缓存**：LRU 策略（最多100个），避免重复数据库查询，缓存键包含搜索参数和过滤状态
3. **数据分离**：
   - JSON 存储工具元数据（id, name, tag, matchClass, files）
   - SQLite 存储用户配置（is_installed, is_favorite, usage_count）
   - 同步时保留用户配置，重建工具索引
4. **模块解耦**：
   - UI 构建（UIBuilder）
   - 数据管理（ToolListManager, MetadataEditor, IconManager）
   - 业务逻辑（HelpViewer, HelpEditor）
   - 各司其职，便于维护
5. **用户隔离**：每个用户独立数据库（UUID），配置互不干扰，支持多数据库切换
6. **单例模式**：搜索窗口使用全局实例，支持复用和热重载
7. **Mixin 模式**：HelpEditorMixin 通过多重继承集成到主应用
8. **观察者模式**：信号槽机制（stateChanged, window_hidden）实现组件通信
9. **工厂模式**：UIBuilder 集中创建和管理 UI 组件
10. **策略模式**：Qt 框架自动检测（PySide6/PySide2），统一接口

## 技术栈

- **GUI 框架**：PySide6 / PySide2（自动检测兼容）
- **数据库**：SQLite3（用户隔离，索引优化）
- **数据格式**：JSON（元数据）、Python dict（内存数据）
- **帮助文档**：Markdown → HTML（markdown 库或内置转换器）
- **图标格式**：PNG（QPixmap 加载和缓存）
- **工具格式**：Python (.py) / Nuke Script (.nk)
- **缓存策略**：LRU（最近最少使用，最多100个搜索结果）
- **搜索技术**：多条件组合查询（精确匹配 + 模糊匹配）
- **文件扫描**：os.scandir（高性能递归扫描）
- **路径管理**：正斜杠统一，跨平台兼容
- **权限控制**：QuanXianSet 模块
